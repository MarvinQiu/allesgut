# AllesGut 现代社交媒体UI设计方案

**设计日期**: 2026-01-29
**设计类型**: UI/UX全面改版
**设计风格**: 受小红书启发的现代社交媒体风格

---

## 一、设计目标

将 AllesGut 从医疗/康复主题的 Soft UI 设计，改造为现代化、年轻化的社交媒体平台。

### 目标用户
泛用户群体（18-45岁），覆盖年轻人、父母、专业人士等不同年龄段。

### 设计关键词
清新、简洁、现代、友好、活力

---

## 二、整体视觉风格

### 设计语言
- 扁平化设计，去除渐变和厚重阴影
- 轻量级社交媒体风格
- 大量留白，强调内容本身
- 简洁的图标和线条

### 视觉原则
1. **简洁优先** - 移除所有装饰性元素，让内容成为焦点
2. **清晰对比** - 使用明确的色彩和字重区分层级
3. **呼吸空间** - 充足的留白和间距
4. **一致性** - 所有组件遵循统一的设计规范

## 五、核心组件设计

### PostCard（内容卡片）

**容器**
- 背景: #FFFFFF
- 圆角: 12px
- 边框: 1px solid #E5E5E5
- 阴影: 0 1px 3px rgba(0,0,0,0.06)
- Hover: 边框变为品牌色，阴影增强

**结构**
```
┌─────────────────────┐
│  [封面图 1/3]       │ ← 右上角多图标识
├─────────────────────┤
│  标题文字最多两行... │
│  #标签1 #标签2      │
│  ○头像 用户名       │
│         ♥128 💬23   │ ← 右对齐
└─────────────────────┘
```

**封面图**
- 圆角: 12px（仅顶部）
- 比例: 保持原始宽高比
- 最小高度: 150px
- 多图标识: 右上角，半透明黑底白字

**内容区**
- 内边距: 12px
- 标题: 14px，font-weight 600，最多2行
- 标签: 圆角12px，浅紫背景，紫蓝文字，最多显示2个
- 作者信息: 头像24px + 昵称12px，间距8px
- 互动数据: 图标18px + 数字12px，灰色

### SearchBar（搜索框）

**尺寸**
- 高度: 44px
- 圆角: 22px
- 全宽减去左右32px边距

**样式**
- 背景: #F7F7F7
- 无边框
- 聚焦时: 2px品牌色边框

**内容**
- 左侧: 放大镜图标16px #999999
- 输入框: 14px占位符
- 右侧: 清除按钮（有内容时显示）

### TagFilter（标签筛选）

**容器**
- 横向滚动，隐藏滚动条
- 左右内边距: 16px

**标签chip**
- 高度: 32px
- 左右内边距: 16px
- 圆角: 16px
- 间距: 8px
- 未选中: #F7F7F7背景，#666666文字
- 选中: 品牌色背景，白色文字，字体加粗

### BottomNavigation（底部导航）

**容器**
- 高度: 56px
- 背景: #FFFFFF
- 顶部边框: 1px #E5E5E5
- 固定底部

**图标**
- 5个图标平均分布
- 尺寸: 24px线性图标
- 未选中: #999999
- 选中: #5B4FFF + 底部3px圆点

**发布按钮**
- 32px圆形
- 品牌色渐变
- 白色+图标
- 突出显示

## 六、页面设计

### 首页（Home）
- 双列瀑布流布局
- 顶部搜索 + Tab切换 + 标签筛选
- 下拉刷新 + 无限滚动
- 骨架屏加载状态

### 发现页（Discover）
- 顶部热门话题横幅（可滑动）
- 分类导航横向滚动
- 双列瀑布流内容
- 支持筛选和搜索

### 发布页（Publish）
- 全屏编辑界面
- 多图上传（最多9张）
- 标题 + 内容编辑
- 添加标签和位置
- 底部发布按钮

### 消息页（Notifications）
- Tab: 互动消息 / 系统通知
- 列表项: 头像 + 用户名 + 操作 + 时间
- 未读红点提示
- 点击进入详情

### 个人中心（Profile）
- 顶部个人信息卡
- 关注/粉丝数据
- 编辑资料按钮
- Tab: 我的发布 / 收藏 / 点赞
- 3列网格布局展示

### 商城页（Mall）
- 顶部商品分类滚动
- 商品卡片: 图 + 标题 + 价格
- 双列网格布局
- 统一卡片样式

## 七、交互动效

### 页面过渡
- 页面切换: 淡入淡出，300ms
- 底部导航: 即时响应，无动画
- 模态弹窗: 底部滑入 + 半透明遮罩

### 交互反馈
- 按钮点击: 缩放0.95，100ms
- 点赞动画: 颜色变化 + 弹跳效果
- 卡片hover: 边框和阴影过渡
- 加载状态: 品牌色转圈

### 滚动优化
- 虚拟滚动（可视区域+缓冲区）
- 图片懒加载（loading="lazy"）
- 骨架屏占位
- 无限滚动自动加载

### 错误处理
- 网络错误: 友好提示 + 重试按钮
- 空状态: 插图 + 引导文案
- 图片失败: 占位图标

## 八、技术实现方案

### 样式架构调整

**Tailwind配置更新**
```javascript
// tailwind.config.js 主要改动
colors: {
  brand: {
    50: '#F5F3FF',
    100: '#E8E6FF',
    500: '#5B4FFF',
    600: '#4A3FE5',
    700: '#3930CC',
  },
  // 移除所有渐变配置
}

// 简化阴影
boxShadow: {
  'card': '0 1px 3px rgba(0,0,0,0.06)',
  'card-hover': '0 2px 8px rgba(0,0,0,0.1)',
}
```

**全局样式类**
```css
.card {
  @apply bg-white rounded-xl border border-gray-200 shadow-card;
}

.btn-primary {
  @apply bg-brand-500 text-white rounded-lg px-4 py-2;
}

.text-main { color: #333333; }
.text-secondary { color: #999999; }
```

### 组件重构策略

**不改变的部分**
- 所有API调用逻辑
- props接口
- 数据处理逻辑
- 路由配置

**需要改变的部分**
- className样式类
- 组件内部DOM结构（必要时）
- CSS样式定义
- 动画效果

**重构优先级**
1. Layout - 全局背景和结构
2. PostCard - 核心内容展示
3. BottomNavigation - 导航体验
4. SearchBar / TagFilter - 筛选交互
5. 其他页面组件

### 图片优化

**格式处理**
- 优先使用WebP，降级到JPG
- 首页缩略图: 300x400规格
- 详情页大图: 按需加载

**加载策略**
- 懒加载: loading="lazy"
- 占位图: 模糊图或纯色
- CDN裁剪: 利用URL参数

### 性能优化

**虚拟滚动**
- 只渲染可视区域+缓冲区
- 使用react-window或自实现

**代码分割**
- 路由级别懒加载
- 大组件动态import

**缓存策略**
- API响应缓存
- 图片浏览器缓存
- Service Worker离线支持

## 十、注意事项

### 开发原则
1. **保持API兼容** - 不修改任何后端接口调用
2. **渐进式改造** - 每个阶段独立测试和上线
3. **功能完整性** - 所有现有功能保持正常
4. **响应式设计** - 适配不同屏幕尺寸

### 风险控制
- Git分支开发，避免影响主线
- 每阶段完成后充分测试
- 保留原样式代码以便回滚
- 逐步灰度发布

### 测试检查清单
- [ ] 所有页面视觉正确
- [ ] 响应式布局正常
- [ ] 交互动画流畅
- [ ] API调用正常
